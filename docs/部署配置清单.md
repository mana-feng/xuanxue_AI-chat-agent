# 部署配置清单

本文档列出将应用部署到实际生产环境时需要修改的所有文件。

## 一、必须创建和配置的文件

### 1. 后端环境变量文件 ⚠️ **必须创建**

**文件路径**: `backend/.env`

**操作步骤**:
1. 复制示例文件：`cp backend/env.example backend/.env`
2. 编辑 `backend/.env`，修改以下配置：

```env
# ============================================
# 后端服务器配置
# ============================================

# 后端服务端口（根据实际情况修改）
PORT=3001

# JWT 密钥（⚠️ 必须修改为强随机字符串，至少32位）
# 生成方式：可以使用 openssl rand -hex 32 或在线工具生成
JWT_SECRET=your_very_long_random_secret_key_at_least_32_characters_long

# Access Token 有效期（默认 30 分钟，可根据需要调整）
ACCESS_TOKEN_EXPIRES_IN=30m

# Refresh Token 有效期（默认 15 天，可根据需要调整）
REFRESH_TOKEN_EXPIRES_DAYS=15

# CORS 允许的来源（主要用于 H5 浏览器端）
# 多个域名用逗号分隔，例如：
# CORS_ORIGINS=https://your-frontend.com,https://h5.your-frontend.com
# 
# 说明：
# - H5（浏览器）：需要配置前端域名，防止未授权网站访问 API
# - 小程序：自动允许，无需配置（微信小程序有独立的域名白名单机制）
# - App：自动允许，无需配置（原生请求不受 CORS 限制）
# 
# 如果只部署小程序或 App，可以留空
# 如果同时部署 H5，生产环境建议配置具体的前端域名
CORS_ORIGINS=https://your-frontend-domain.com,https://h5.your-frontend-domain.com

# ============================================
# MySQL 数据库配置（⚠️ 必须修改为实际数据库信息）
# ============================================

DB_HOST=localhost
DB_PORT=3306
DB_USER=your_db_user
DB_PASS=your_db_password
DB_NAME=bazi_app
```

**重要提示**:
- ⚠️ `JWT_SECRET` 必须使用强随机字符串，建议使用 `openssl rand -hex 32` 生成
- ⚠️ `DB_PASS` 必须设置为实际的数据库密码
- ⚠️ `CORS_ORIGINS` 配置说明：
  - 如果只部署小程序或 App：可以留空（小程序和 App 不受 CORS 限制）
  - 如果部署 H5（浏览器）：必须配置前端域名，防止未授权网站访问
- ⚠️ `.env` 文件不要提交到 Git 仓库（已在 .gitignore 中）
- ⚠️ 默认管理员账号通过 `init-mysql.sql` 脚本创建，如需修改请编辑该文件

---

### 2. 前端环境变量文件 ⚠️ **必须创建**

**文件路径**: `frontend/.env.production`

**操作步骤**:
1. 复制示例文件：`cp frontend/env.example frontend/.env.production`
2. 编辑 `frontend/.env.production`，修改以下配置：

```env
# ============================================
# 前端环境变量配置（生产环境）
# ============================================
# 重要提示：前端配置文件中不要包含任何数据库信息！
# 前端仅需要知道后端API的地址
# ============================================

# 后端API基础地址（⚠️ 必须修改为实际的后端API地址）
# 生产环境示例：https://api.yourdomain.com
# 如果使用Nginx反向代理，使用HTTPS域名
VITE_API_BASE_URL=https://api.yourdomain.com
```

**重要提示**:
- ⚠️ `VITE_API_BASE_URL` 必须设置为实际的后端API地址
- ⚠️ 生产环境必须使用 HTTPS（`https://`）
- ⚠️ 前端配置文件中**不要**包含数据库连接信息
- ⚠️ 前端配置文件中**不要**包含JWT密钥等敏感信息

## 二、可能需要检查的文件

### 3. 数据库初始化脚本（通常不需要修改）

**文件路径**: `backend/init-mysql.sql`

**检查项**:
- 数据库名称：默认为 `bazi_app`，如果需要修改，需要同时修改：
  - `backend/.env` 中的 `DB_NAME`
  - `backend/init-mysql.sql` 中的数据库名称

**管理员账号配置**:
- 默认管理员账号通过 `backend/init-mysql.sql` 脚本创建
- 如需修改管理员账号信息，请编辑 `init-mysql.sql` 文件后重新执行初始化
- 也可以通过后台管理界面手动创建管理员账号

⚠️ **安全提示**: 
- 生产环境建议修改 `init-mysql.sql` 中的默认密码 hash
- 首次登录后建议立即修改密码
- 密码 hash 生成方式：`node -e "const bcrypt = require('bcryptjs'); console.log(bcrypt.hashSync('your_password', 10));"`

---

### 4. 前端配置文件（通常不需要修改）

**文件路径**: `frontend/src/config/config.ts`

**说明**: 
- 此文件会自动从环境变量 `VITE_API_BASE_URL` 读取API地址
- 通常不需要手动修改
- 如果构建时环境变量未正确注入，会回退到 `http://localhost:3001`

## 三、部署后需要配置的内容

### 5. 数据库初始化

**操作步骤**:
```bash
# 1. 登录 MySQL
mysql -u your_db_user -p

# 2. 创建数据库（如果不存在）
CREATE DATABASE IF NOT EXISTS bazi_app CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 3. 导入初始化脚本
mysql -u your_db_user -p bazi_app < backend/init-mysql.sql
```

或者，如果后端服务已启动，表结构会自动创建（通过 `autoInitMySQLTables` 函数）。

---

### 6. 后台管理配置（部署后通过Web界面配置）

部署完成后，登录后台管理系统配置以下内容：

1. **邮箱SMTP配置**（用于发送验证码）
   - 路径: 后台管理 → 邮箱配置
   - 需要配置：SMTP服务器、端口、账号、密码、发件人邮箱

2. **LLM配置**（用于AI聊天功能）
   - 路径: 后台管理 → LLM配置
   - 需要配置：供应商、API地址、API密钥、模型名称

## 五、快速部署命令

### 后端部署

```bash
# 1. 进入后端目录
cd backend

# 2. 创建环境变量文件
cp env.example .env

# 3. 编辑 .env 文件（使用你喜欢的编辑器）
nano .env
# 或
vim .env

# 4. 安装依赖
npm install

# 5. 启动服务（开发环境）
node app.js

# 或使用 PM2（生产环境推荐）
pm2 start app.js --name bazi-api
pm2 save
```

### 前端部署

```bash
# 1. 进入前端目录
cd frontend

# 2. 创建生产环境变量文件
cp env.example .env.production

# 3. 编辑 .env.production 文件
nano .env.production
# 或
vim .env.production

# 4. 安装依赖
npm install

# 5. 构建前端（根据目标平台选择）
# H5 构建
npm run build:h5

# 微信小程序构建
npm run build:mp-weixin

# App 构建
npm run build:app
```

## 六、常见问题

### Q1: 如何生成强随机 JWT_SECRET？

```bash
# 方法1: 使用 openssl
openssl rand -hex 32

# 方法2: 使用 Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# 方法3: 使用在线工具
# 访问 https://www.random.org/strings/ 生成随机字符串
```

### Q2: 前端构建后API地址不对？

**原因**: 环境变量未正确注入到构建产物中。

**解决方案**:
1. 确保 `frontend/.env.production` 文件存在且配置正确
2. 确保构建命令正确：`npm run build:h5`
3. 检查构建后的代码中是否包含正确的API地址

### Q3: CORS 错误？

**原因**: 后端 `CORS_ORIGINS` 配置不正确。

**解决方案**:
1. 检查 `backend/.env` 中的 `CORS_ORIGINS` 配置
2. 确保前端域名在允许列表中
3. 确保域名格式正确（包含协议，如 `https://`）

### Q4: 数据库连接失败？

**检查项**:
1. MySQL 服务是否启动
2. `backend/.env` 中的数据库配置是否正确
3. 数据库用户权限是否正确
4. 防火墙是否允许连接

## 七、文件修改总结

| 文件路径 | 操作 | 必须性 | 说明 |
|---------|------|--------|------|
| `backend/.env` | 创建并配置 | ⚠️ **必须** | 后端环境变量配置 |
| `frontend/.env.production` | 创建并配置 | ⚠️ **必须** | 前端生产环境变量配置 |
| `backend/init-mysql.sql` | 检查 | 可选 | 数据库初始化脚本（通常不需要修改） |
| `frontend/src/config/config.ts` | 检查 | 可选 | 前端配置文件（通常不需要修改） |

## 八、安全建议

1. **环境变量安全**
   - 不要将 `.env` 文件提交到 Git
   - 使用强密码和随机密钥
   - 定期轮换密钥和密码

2. **数据库安全**
   - 使用强密码
   - 限制数据库远程访问
   - 定期备份数据库

3. **HTTPS配置**
   - 生产环境必须使用 HTTPS
   - 使用 Let's Encrypt 免费证书
   - 配置 HSTS 等安全头

4. **防火墙配置**
   - 只开放必要端口（80, 443, 3001）
   - 限制数据库端口访问（仅允许本地或指定IP）
