# 安全架构说明

## 前后端分离架构

本项目采用完全的前后端分离架构，确保数据安全和隐私保护。

### 核心原则

1. **排盘计算完全在前端完成**
   - 所有排盘计算使用 `lunar-javascript` 库在前端本地执行
   - 后端不进行任何排盘计算，只负责数据存储和验证
   - 用户数据不会因为排盘计算而暴露给后端

2. **后端只处理必要的数据**
   - 后端只存储用户提供的基本信息（姓名、性别、出生时间）
   - 可选的 `rawPayload` 仅用于前端数据恢复，不用于任何计算
   - 后端不依赖 `rawPayload` 进行任何业务逻辑

3. **数据传输安全**
   - 所有 API 请求通过 HTTPS 传输（生产环境）
   - 使用 JWT Token 进行身份认证
   - 敏感操作（POST/PUT/DELETE）支持 API 签名验证
   - 请求数据经过清理，移除敏感信息

## 安全措施

### 1. API 签名机制

- **目的**：防止请求被篡改和重放攻击
- **实现**：
  - 每个请求包含时间戳（timestamp）和随机数（nonce）
  - 使用 HMAC-SHA256 生成签名
  - 后端强制验证签名和时间戳有效性
  - 防止重放攻击（nonce 只能使用一次）
  - **强制模式**：所有使用签名验证的路由都必须提供有效签名

- **配置**：
  - 必须设置 `API_SIGNATURE_SECRET` 环境变量
  - 所有敏感操作（POST/PUT/DELETE）都强制要求签名

### 2. 数据验证和清理

- **输入验证**：
  - 所有用户输入都经过验证和清理
  - 防止 SQL 注入和 XSS 攻击
  - 数据类型和格式验证

- **数据清理**：
  - 移除敏感字段（密码、token 等）
  - 限制数据大小
  - 清理特殊字符

### 3. 身份认证

- **JWT Token 机制**：
  - Access Token：短期有效（默认 30 分钟）
  - Refresh Token：长期有效（默认 15 天）
  - 自动刷新机制，无需用户重新登录

- **设备绑定**：
  - 每个请求包含设备 ID
  - 可用于异常检测和访问控制

### 4. 请求频率限制

- **目的**：防止暴力攻击和资源滥用
- **实现**：
  - 基于 IP 和用户 ID 的频率限制
  - 超过限制的请求会被阻止一段时间

### 5. 安全日志

- **记录内容**：
  - 可疑请求（SQL 注入、XSS 攻击尝试）
  - 认证失败
  - 频率限制触发
  - 服务器错误

- **隐私保护**：
  - 不记录敏感数据（密码、完整请求体）
  - 只记录必要的元数据

## 数据存储

### 数据库安全

- **参数化查询**：所有数据库操作使用参数化查询，防止 SQL 注入
- **数据加密**：敏感数据在存储前进行加密（如需要）
- **访问控制**：用户只能访问自己的数据

### 数据最小化

- **只存储必要数据**：
  - 用户基本信息（姓名、性别、出生时间）
  - 可选的排盘结果快照（用于前端恢复）
  - 不存储敏感的个人信息

## 前端安全

### 1. 数据本地计算

- 所有排盘计算在前端完成
- 不向服务器发送完整的计算过程
- 计算结果只在用户本地显示

### 2. 敏感信息保护

- 不在日志中记录敏感信息
- 不在网络请求中发送敏感数据
- 使用安全存储（secureStorage）保存 token

### 3. 代码混淆

- 生产环境代码经过混淆和压缩
- 减少代码被逆向工程的风险

## 部署建议

### 生产环境配置

1. **环境变量**：
   ```bash
   NODE_ENV=production
   JWT_SECRET=<强随机字符串>
   API_SIGNATURE_SECRET=<强随机字符串>
   API_SIGNATURE_SECRET=<强随机字符串>
   CORS_ORIGINS=<允许的域名列表>
   ```

2. **HTTPS**：
   - 必须启用 HTTPS
   - 使用有效的 SSL 证书
   - 配置 HSTS 头

3. **数据库**：
   - 使用强密码
   - 限制数据库访问 IP
   - 定期备份

4. **服务器**：
   - 定期更新系统和依赖
   - 配置防火墙规则
   - 启用日志监控

## 安全审计

### 定期检查项

1. **依赖更新**：定期更新 npm 包，修复安全漏洞
2. **日志审查**：检查安全日志，发现异常访问
3. **访问控制**：审查用户权限和访问记录
4. **数据备份**：确保数据备份的完整性和可恢复性
