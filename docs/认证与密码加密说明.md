# 认证与密码加密说明

## 一、后端密码存储加密

### 1.1 密码加密方式
- **加密算法**：使用 `bcrypt` 进行单向哈希加密
- **加密强度**：使用 `bcrypt.hashSync(password, 10)`，salt rounds = 10
- **存储字段**：数据库中存储的是 `password_hash`，**不存储明文密码**

### 1.2 密码验证流程
```javascript
// 后端验证密码（server/app.js）
const match = bcrypt.compareSync(String(password), row.password_hash);
if (!match) {
    return res.status(401).json({ error: '账号或密码错误' });
}
```

### 1.3 密码存储位置
- **数据库表**：`users` 表
- **字段名**：`password_hash` (VARCHAR(255))
- **示例**：`$2a$10$xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

## 二、前端登录缓存加密

### 2.1 缓存加密方式
- **加密算法**：AES-256-CBC（使用 crypto-js）
- **存储位置**：uni-app 本地存储（`uni.setStorageSync`）
- **加密内容**：登录信息（username, email, token, role）
- **过期时间**：15 天（自动过期）

### 2.2 加密存储工具
- **文件位置**：`src/utils/secureStorage.ts`
- **主要方法**：
  - `setSecureItem(key, value, ttl?)` - 加密存储，带过期时间
  - `getSecureItem(key)` - 解密读取，自动检查过期
  - `removeSecureItem(key)` - 删除加密数据

### 2.3 存储的数据结构
```typescript
{
  username: string | null;
  email: string;
  token: string;      // JWT token
  role: 'user' | 'admin';
  _expires?: number;  // 过期时间戳（内部使用）
}
```

## 三、前后端互相校验流程

### 3.1 登录流程

#### 前端 → 后端
1. 用户输入邮箱/用户名和密码
2. 前端发送 POST 请求到 `/api/login`
3. **密码以明文传输**（HTTPS 环境下安全）

#### 后端验证
1. 从数据库查询用户：`SELECT id, email, username, password_hash, role FROM users WHERE email = ? OR username = ?`
2. 使用 `bcrypt.compareSync` 比较密码
3. 如果匹配，生成 JWT token：
   ```javascript
   const token = jwt.sign({ uid: row.id }, JWT_SECRET, { expiresIn: '7d' });
   ```
4. 返回 token 和用户信息

#### 前端存储
1. 收到 token 后，调用 `user_store.setAuth()` 
2. 使用 `setSecureItem('auth', {...})` 加密存储登录信息
3. 缓存有效期 15 天

### 3.2 请求认证流程

#### 前端发送请求
1. 调用 `request(url, { needAuth: true })`
2. 从加密缓存读取 token：`getSecureItem('auth')`
3. 在请求头中添加：`Authorization: Bearer <token>`

#### 后端验证
1. `authMiddleware` 中间件拦截请求
2. 从请求头提取 token：`Authorization: Bearer <token>`
3. 使用 `jwt.verify(token, JWT_SECRET)` 验证 token
4. 如果有效，提取 `uid` 并设置 `req.user = { id: decoded.uid }`
5. 如果无效或过期，返回 401 错误

#### 前端处理 401
1. 检测到 401 响应
2. 自动清除登录缓存：`removeSecureItem('auth')`
3. 提示用户重新登录

### 3.3 Token 有效期
- **后端 JWT 有效期**：7 天（`expiresIn: '7d'`）
- **前端缓存有效期**：15 天
- **说明**：前端缓存过期时间比 JWT 长，但实际使用时仍以 JWT 有效期为准

## 四、安全特性

### 4.1 密码安全
- ✅ 密码使用 bcrypt 单向哈希，无法逆向
- ✅ 数据库不存储明文密码
- ✅ 密码传输建议使用 HTTPS（生产环境）

### 4.2 Token 安全
- ✅ JWT token 使用密钥签名，防止篡改
- ✅ Token 包含过期时间，自动失效
- ✅ 前端缓存加密存储，增加安全性

### 4.3 缓存安全
- ✅ 登录信息 AES 加密存储
- ✅ 自动过期机制（15 天）
- ✅ Token 过期后自动清除缓存

## 五、相关文件

- `server/app.js` - 登录接口、密码加密、JWT 验证

### 前端
- `src/store/user.ts` - 用户状态管理，使用加密存储

- `src/api/index.ts` - API 请求封装，自动添加 token
- `src/pages/auth/auth.nvue` - 登录/注册页面

## 六、注意事项

1. **生产环境**：
   - 必须使用 HTTPS 传输密码
   - JWT_SECRET 应使用强随机字符串
   - 定期更新 JWT_SECRET 会导致所有 token 失效

2. **密码策略**：
   - 后端有密码验证（`SecurityUtils.validatePassword`）
   - 建议前端也添加密码强度提示

3. **Token 刷新**：
   - 当前未实现 token 刷新机制
   - Token 过期后需要重新登录

4. **多设备登录**：
   - 当前每个设备独立 token
   - 未实现设备管理功能
