# 部署指南

本文档说明如何部署前后端完全分离的应用。

## 架构说明

- **前端** (`src/`): uni-app 应用，支持 Web、App、微信小程序
  - 仅通过 HTTP API 与后端通信
  - 不包含任何数据库连接信息
  - 配置：仅需要后端API地址（`VITE_API_BASE_URL`）

- **后端** (`server/`): Node.js + Express API 服务
  - 提供 RESTful API 和 WebSocket 服务
  - 所有数据存储在 MySQL 数据库
  - 配置：数据库连接、JWT密钥、CORS等

## 一、后端部署

### 1. 环境要求

- Node.js >= 16.x
- MySQL >= 5.7 或 >= 8.0
- 开放端口：3001（或自定义端口）

### 2. 上传后端代码

将以下文件/目录上传到服务器：

```
server/
  ├── app.js
  ├── db.js
  ├── config-service.js
  ├── security.js
  ├── security-middleware.js
  └── init-mysql.sql
package.json（仅需要 server 目录下的依赖）
```

### 3. 安装依赖

```bash
cd /path/to/your/server
npm install express mysql2 jsonwebtoken bcryptjs cookie-parser cors express-rate-limit ws node-fetch dotenv nodemailer
```

### 4. 配置环境变量

在服务器上创建 `.env` 文件：

```env
# 后端服务端口
PORT=3001

# JWT 密钥（必须修改为随机字符串）
JWT_SECRET=your_random_secret_key_here_change_me

# Access Token 有效期（默认 30 分钟）
ACCESS_TOKEN_EXPIRES_IN=30m

# Refresh Token 有效期（默认 15 天）
REFRESH_TOKEN_EXPIRES_DAYS=15

# CORS 允许的来源（生产环境建议配置具体的前端域名）
# 多个域名用逗号分隔，例如：https://your-frontend.com,https://h5.your-frontend.com
# 留空则允许所有来源（仅开发环境使用）
CORS_ORIGINS=https://your-frontend-domain.com,https://h5.your-frontend-domain.com

# MySQL 数据库配置（仅后端需要）
DB_HOST=localhost
DB_PORT=3306
DB_USER=your_db_user
DB_PASS=your_db_password
DB_NAME=bazi_app
```

### 5. 初始化数据库

```bash
# 登录 MySQL
mysql -u your_db_user -p

# 创建数据库
CREATE DATABASE IF NOT EXISTS bazi_app CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 导入初始化脚本
mysql -u your_db_user -p bazi_app < server/init-mysql.sql
```

### 6. 启动后端服务

#### 方式一：使用 PM2（推荐）

```bash
# 安装 PM2
npm install -g pm2

# 启动服务
pm2 start server/app.js --name bazi-api

# 设置开机自启
pm2 startup
pm2 save

# 查看日志
pm2 logs bazi-api
```

#### 方式二：使用 systemd（Linux）

创建 `/etc/systemd/system/bazi-api.service`：

```ini
[Unit]
Description=Bazi API Server
After=network.target mysql.service

[Service]
Type=simple
User=your_user
WorkingDirectory=/path/to/your/server
Environment=NODE_ENV=production
ExecStart=/usr/bin/node server/app.js
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl enable bazi-api
sudo systemctl start bazi-api
sudo systemctl status bazi-api
```

### 7. 配置 Nginx 反向代理（可选，推荐）

```nginx
server {
    listen 80;
    server_name api.yourdomain.com;

    location / {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 8. 配置 HTTPS（生产环境必须）

```bash
sudo apt-get install certbot python3-certbot-nginx
sudo certbot --nginx -d api.yourdomain.com
```

## 二、前端部署

### 1. 配置前端环境变量

在项目根目录创建 `.env.production` 文件（**仅包含API地址，不包含数据库信息**）：

```env
# 前端构建时注入的后端API地址
# 生产环境：使用HTTPS域名
VITE_API_BASE_URL=https://api.yourdomain.com
```

**重要**: 
- 前端配置文件中**不要**包含任何数据库连接信息
- 前端配置文件中**不要**包含JWT密钥等敏感信息
- 前端仅需要知道后端API的域名和端口

### 2. 构建前端

#### Web (H5) 构建

```bash
# 安装依赖
npm install

# 构建H5版本
npm run build:h5
```

构建完成后，代码在 `dist/build/h5/` 目录。

#### 微信小程序构建

```bash
# 构建微信小程序
npm run build:mp-weixin
```

构建完成后，代码在 `dist/build/mp-weixin/` 目录。

#### App 构建

```bash
# 构建App版本
npm run build:app
```

### 3. 部署前端

#### Web (H5) 部署

将 `dist/build/h5/` 目录的内容上传到Web服务器。

Nginx配置示例：

```nginx
server {
    listen 80;
    server_name your-frontend-domain.com;

    root /path/to/dist/build/h5;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

#### 微信小程序部署

1. 打开 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 选择"导入项目"
3. 选择 `dist/build/mp-weixin/` 目录
4. 填写 AppID（与 `src/manifest.json` 中的一致）
5. 点击"编译"
6. 测试功能是否正常
7. 点击"上传"提交代码
8. 在微信公众平台提交审核和发布

### 4. 配置微信小程序 request 合法域名

在微信公众平台配置：

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入您的小程序后台
3. 开发 → 开发管理 → 开发设置
4. 服务器域名 → request合法域名
5. 添加您的后端 API 域名（如：`https://api.yourdomain.com`）
6. 如果需要 WebSocket，在 socket合法域名 中添加 WebSocket 地址（如：`wss://api.yourdomain.com`）

**注意**：
- 域名必须使用 HTTPS（生产环境）
- 域名必须已备案（如果使用国内服务器）
- 开发时可以在微信开发者工具中勾选"不校验合法域名"

## 四、安全建议

### 后端安全

1. **修改 JWT_SECRET**：使用强随机字符串
2. **配置 HTTPS**：使用 Nginx + Let's Encrypt
3. **限制 CORS**：生产环境配置具体的前端域名
4. **数据库安全**：
   - 使用强密码
   - 限制远程访问（仅允许本地或指定IP）
   - 定期备份数据库
5. **防火墙配置**：只开放必要端口（80, 443, 3001）

### 前端安全

1. **启用域名校验**：生产环境设置 `urlCheck: true`（小程序）
2. **隐藏敏感信息**：不要在代码中硬编码 API Key 等
3. **使用 HTTPS**：所有 API 请求必须使用 HTTPS
4. **定期更新**：及时更新依赖包

## 五、常见问题

### 1. 前端无法连接后端

**检查项**：
- 后端服务是否正常运行
- `VITE_API_BASE_URL` 配置是否正确
- 网络是否可达
- CORS 配置是否正确（如果是Web）
- 微信小程序是否配置了 request合法域名

### 2. CORS 错误（Web）

**解决方案**：
- 检查后端 `.env` 中的 `CORS_ORIGINS` 配置
- 确保前端域名在允许列表中
- 检查后端是否正常启动

### 3. 数据库连接失败

**检查项**：
- MySQL 服务是否启动
- `.env` 中的数据库配置是否正确
- 数据库用户权限是否正确
- 防火墙是否允许连接

## 六、快速部署脚本示例

### 后端部署脚本 (`deploy-backend.sh`)

```bash
#!/bin/bash

echo "开始部署后端..."

# 1. 拉取最新代码
git pull

# 2. 安装依赖
npm install

# 3. 重启服务
pm2 restart bazi-api

echo "后端部署完成！"
```

### 前端部署脚本 (`deploy-frontend.sh`)

```bash
#!/bin/bash

echo "开始部署前端..."

# 1. 拉取最新代码
git pull

# 2. 安装依赖
npm install

# 3. 构建前端
npm run build:h5

# 4. 上传到Web服务器（根据实际情况修改）
# scp -r dist/build/h5/* user@server:/path/to/web/root/

echo "前端部署完成！"
```
