## 🐛 发现的问题

### 1. 数据库配置警告（已修复）

**问题**：
```
Ignoring invalid configuration option passed to Connection: max_allowed_packet
Ignoring invalid configuration option passed to Connection: reconnect
```

**原因**：
- `max_allowed_packet` 不是 MySQL2 连接池的有效配置选项
- `reconnect` 不是 MySQL2 连接池的有效配置选项

**修复**：
- ✅ 已从 `backend/db.js` 中移除这两个无效配置选项
- ✅ 添加了注释说明这些选项的正确使用方式

### 2. 数据库连接失败（需要用户配置）

**问题**：
```
Access denied for user 'root'@'localhost' (using password: YES)
```

**原因**：
- 数据库密码配置不正确或数据库未启动
- 这是正常的，需要用户根据实际情况配置

**解决方案**：
1. 编辑 `backend/.env` 文件，配置正确的数据库信息：
   ```env
   DB_HOST=localhost
   DB_PORT=3306
   DB_USER=root
   DB_PASS=your_actual_password
   DB_NAME=bazi_app
   ```

2. 确保 MySQL 服务已启动

3. 创建数据库（如果不存在）：
   ```sql
   CREATE DATABASE IF NOT EXISTS bazi_app CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

4. 初始化数据库表结构：
   ```bash
   mysql -u root -p bazi_app < backend/init-mysql.sql
   ```

## 🚀 启动步骤

### 1. 配置数据库

编辑 `backend/.env` 文件，配置数据库信息。

### 2. 初始化数据库

```bash
# 创建数据库
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS bazi_app CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 初始化表结构
mysql -u root -p bazi_app < backend/init-mysql.sql
```

### 3. 启动后端

```bash
cd backend
npm start
```

### 4. 启动前端（新终端）

```bash
cd frontend
npm run dev:h5
```

### 5. 或使用启动脚本（Windows）

```bash
start-dev.bat
```
