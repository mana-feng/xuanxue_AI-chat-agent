# 构建问题说明

## 构建状态

✅ **构建成功** - 项目可以正常打包为 Android 应用

⚠️ **存在大量 nvue CSS 兼容性警告** - 这些警告不会阻止打包，但可能影响应用在 Android 上的显示效果

## 主要问题分类

### 1. nvue 不支持的 CSS 属性

nvue 是 uni-app 的原生渲染引擎，只支持有限的 CSS 属性。以下属性在 nvue 中**不支持**：

#### 布局相关
- ❌ `display: grid` - 只支持 `flex`
- ❌ `display: block` - 只支持 `flex`
- ❌ `display: inline-block` - 只支持 `flex`
- ❌ `gap` - 不支持，需要使用 `margin` 替代
- ❌ `grid-template-columns` - 不支持 Grid 布局
- ❌ `grid-column` - 不支持 Grid 布局

#### 尺寸相关
- ❌ `min-width` / `max-width` - 不支持
- ❌ `min-height` / `max-height` - 不支持
- ❌ `box-sizing` - 不支持

#### 函数和变量
- ❌ `clamp()` - 不支持，需要使用固定值
- ❌ `var()` - 不支持 CSS 变量
- ❌ `calc()` - 部分支持，但有限制

#### 动画和过渡
- ❌ `@keyframes` - 不支持，需要使用 uni-app 动画 API
- ❌ `animation` - 不支持
- ❌ `transition: all` - 不支持，需要指定具体属性

#### 其他
- ❌ `:root` 选择器 - 不支持
- ❌ `::before` / `::after` 伪元素 - 不支持
- ❌ `::-webkit-scrollbar` - 不支持
- ❌ `filter` / `backdrop-filter` - 不支持
- ❌ `cursor` - 不支持
- ❌ `white-space` - 不支持
- ❌ `overflow-x` / `overflow-y` - 不支持
- ❌ `user-select` - 不支持

### 2. 值类型限制

某些属性只接受特定类型的值：

- ❌ `width: auto` - 只支持数字和像素值
- ❌ `margin: auto` - 只支持数字和像素值
- ❌ `flex: 0 0 auto` - 只支持数字值
- ❌ `border-radius: var(--radius-md)` - 只支持数字和像素值
- ❌ `!important` - 在某些属性上不支持

### 3. 选择器限制

nvue 只支持类名选择器：

- ❌ `:root` - 不支持
- ❌ `@keyframes 0%` - 不支持
- ❌ `.class table` - 不支持后代选择器
- ❌ `.class uni-text` - 不支持组件选择器

## 解决方案

### 方案一：使用条件编译（推荐）

为 nvue 页面创建专门的样式：

```vue
<style lang="scss">
/* #ifndef APP-PLUS-NVUE */
/* H5 和小程序样式 */
.container {
  display: grid;
  gap: 16px;
}
/* #endif */

/* #ifdef APP-PLUS-NVUE */
/* nvue 样式 */
.container {
  display: flex;
  flex-direction: column;
}
.container-item {
  margin-bottom: 16px;
}
/* #endif */
</style>
```

### 方案二：使用 Flex 布局替代 Grid

将 Grid 布局改为 Flex 布局：

```scss
// 原代码（不支持）
.stats-grid {
  display: grid;
  grid-template-columns: repeat(2, 1fr);
  gap: 20rpx;
}

// 修改为（支持）
.stats-grid {
  display: flex;
  flex-direction: row;
  flex-wrap: wrap;
}
.stats-item {
  width: calc(50% - 10rpx);
  margin-right: 20rpx;
  margin-bottom: 20rpx;
}
```

### 方案三：使用 margin 替代 gap

```scss
// 原代码（不支持）
.flex-container {
  display: flex;
  gap: 16px;
}

// 修改为（支持）
.flex-container {
  display: flex;
}
.flex-item {
  margin-right: 16px;
}
.flex-item:last-child {
  margin-right: 0;
}
```

### 方案四：使用固定值替代 clamp()

```scss
// 原代码（不支持）
.title {
  font-size: clamp(16px, 2vw, 20px);
}

// 修改为（支持）
.title {
  font-size: 18px; /* 或使用 rpx */
}
```

### 方案五：使用 uni-app 动画 API 替代 CSS 动画

```javascript
// 原代码（不支持）
// CSS: @keyframes pulse { ... }

// 修改为（支持）
// 使用 uni.createAnimation() API
const animation = uni.createAnimation({
  duration: 2000,
  timingFunction: 'ease-in-out'
})
```

## 优先级建议

### 高优先级（影响功能）
1. `display: grid` → 改为 `flex` 布局
2. `gap` → 改为 `margin`
3. `width: auto` → 改为具体数值

### 中优先级（影响显示）
1. `clamp()` → 改为固定值
2. `min-width` / `max-width` → 使用 `width` 和 `flex`
3. CSS 变量 → 改为具体值

### 低优先级（不影响核心功能）
1. `@keyframes` → 使用 uni-app 动画 API（可选）
2. `filter` / `backdrop-filter` → 移除或使用图片替代
3. `cursor` → 移除（移动端不需要）
