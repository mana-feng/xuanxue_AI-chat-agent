# 项目结构说明

## 目录结构

本项目采用**完全前后端分离**的架构，目录结构清晰，前后端完全隔离。

```
AI-xuanxue/
├── frontend/                    # 前端项目（完全独立）
│   ├── src/                    # 前端源代码
│   │   ├── api/                # API调用封装
│   │   ├── pages/               # 页面
│   │   ├── components/         # 组件
│   │   ├── store/              # 状态管理
│   │   ├── utils/              # 工具函数
│   │   └── config/             # 配置文件（仅API地址）
│   ├── dist/                   # 构建产物（开发/生产）
│   ├── unpackage/              # uni-app 打包产物
│   ├── scripts/                # 构建脚本
│   ├── package.json            # 前端依赖管理
│   ├── vite.config.ts          # Vite 配置
│   ├── tsconfig.json           # TypeScript 配置
│   ├── androidPrivacy.json     # Android隐私配置
│   └── README.md               # 前端项目说明
│
├── backend/                    # 后端项目（完全独立）
│   ├── app.js                  # 主应用入口
│   ├── db.js                   # 数据库连接
│   ├── config/                 # 配置文件
│   ├── routes/                 # 路由
│   ├── middleware/             # 中间件
│   ├── services/               # 业务服务
│   ├── utils/                  # 工具函数
│   ├── database/               # 数据库初始化
│   ├── websocket/              # WebSocket服务
│   ├── package.json            # 后端依赖管理
│   ├── env.example             # 环境变量示例
│   └── README.md               # 后端项目说明
│
├── docs/                       # 项目文档
│   ├── API接口文档.md          # API接口说明
│   ├── 部署指南.md             # 部署流程
│   ├── 安全架构说明.md         # 安全架构
│   └── ...                     # 其他文档
│
├── package.json                # 根目录统一管理脚本
├── README.md                   # 项目主说明
├── start-dev.bat               # 开发环境启动脚本（Windows）
├── stop-dev.bat                # 停止服务脚本（Windows）
└── .gitignore                  # Git忽略配置
```

## 前后端隔离原则

### 1. 目录隔离
- ✅ 前端所有文件在 `frontend/` 目录
- ✅ 后端所有文件在 `backend/` 目录
- ✅ 根目录只保留统一管理脚本和文档

### 2. 依赖隔离
- ✅ 前端有自己的 `node_modules/` 和 `package.json`
- ✅ 后端有自己的 `node_modules/` 和 `package.json`
- ✅ 根目录不包含 `node_modules/`

### 3. 构建产物隔离
- ✅ 前端构建产物在 `frontend/dist/` 和 `frontend/unpackage/`
- ✅ 后端不包含前端构建产物
- ✅ 根目录不包含构建产物

### 4. 配置隔离
- ✅ 前端配置在 `frontend/` 目录（仅API地址）
- ✅ 后端配置在 `backend/` 目录（数据库、密钥等）
- ✅ 配置文件不互相引用

### 5. 通信方式
- ✅ 前后端仅通过 HTTP/HTTPS API 通信
- ✅ 前端通过配置文件指定后端API地址
- ✅ 不共享代码、不共享数据库连接

## 文件说明

### 根目录文件

| 文件 | 说明 | 是否必需 |
|------|------|---------|
| `package.json` | 统一管理脚本（install:all, start:frontend等） | 可选 |
| `README.md` | 项目主说明文档 | 必需 |
| `start-dev.bat` | Windows开发环境启动脚本 | 可选 |
| `stop-dev.bat` | Windows停止服务脚本 | 可选 |
| `.gitignore` | Git忽略配置 | 必需 |

### 前端目录 (`frontend/`)

| 目录/文件 | 说明 |
|-----------|------|
| `src/` | 前端源代码 |
| `dist/` | 构建产物（开发/生产） |
| `unpackage/` | uni-app 打包产物 |
| `package.json` | 前端依赖管理 |
| `androidPrivacy.json` | Android隐私配置 |

### 后端目录 (`backend/`)

| 目录/文件 | 说明 |
|-----------|------|
| `app.js` | 主应用入口 |
| `routes/` | API路由 |
| `middleware/` | 中间件 |
| `services/` | 业务服务 |
| `package.json` | 后端依赖管理 |
| `.env` | 环境变量（不提交到Git） |

## 开发规范

### 1. 文件位置
- ✅ 前端相关文件 → `frontend/`
- ✅ 后端相关文件 → `backend/`
- ✅ 文档 → `docs/`
- ✅ 根目录只保留管理脚本

### 2. 依赖管理
- ✅ 前端依赖在 `frontend/package.json`
- ✅ 后端依赖在 `backend/package.json`
- ✅ 不要跨目录引用依赖

### 3. 配置管理
- ✅ 前端配置仅包含API地址
- ✅ 后端配置包含数据库、密钥等敏感信息
- ✅ 使用环境变量，不要硬编码

### 4. 构建和部署
- ✅ 前端独立构建和部署
- ✅ 后端独立部署
- ✅ 可以部署到不同的服务器

## 快速开始

### 安装依赖

```bash
# 方式1：使用根目录统一脚本
npm run install:all

# 方式2：分别安装
cd frontend && npm install && cd ..
cd backend && npm install
```

### 启动开发环境

```bash
# Windows
start-dev.bat

# 或手动启动
# 终端1 - 后端
cd backend && npm start

# 终端2 - 前端
cd frontend && npm run dev:h5
```

## 注意事项

1. **不要跨目录引用**：前端不要直接引用后端代码，反之亦然
2. **配置文件分离**：前后端配置文件完全独立
3. **构建产物隔离**：前端构建产物不要放在根目录
4. **依赖隔离**：根目录不应该有 `node_modules/`
5. **文档集中**：所有文档放在 `docs/` 目录
